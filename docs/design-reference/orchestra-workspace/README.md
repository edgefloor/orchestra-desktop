# Orchestra workspace reference

This archive freezes the deployed workspace wireframe as directional design input for the Orchestra desktop fork. It is not a runtime, product-data, or pixel-parity authority.

The reference was captured from `https://orchestra.demystify.hu/orchestra-workspace` on 2026-07-17. `reference.json` records the fetched HTML digest and seals the approved screenshots. The HTML itself is not vendored: its digest identifies the exact inspected deployment without importing prototype code into the product fork.

## Precedence

When the reference conflicts with the product, use this order:

1. Native Codex and desktop behavior
2. Retained T3Code capabilities
3. A coherent Orchestra fusion
4. Reference pixels

The reference contributes information architecture, density, hierarchy, brand direction, and responsive intent. It does not authorize detached workflow execution, alternate control planes, fixture-only actions, or replacement of mature native interactions.

## Approved viewports

- `desktop-1280x720.jpg` is the approved desktop composition. Persistent project/task navigation, the task workspace, retained panel tabs, and task controls should remain simultaneously legible at this class of viewport.
- `narrow-desktop-900x506.jpg` is the approved narrow-desktop composition captured by the in-app browser. At this class of viewport, the task remains the primary focus; panels may collapse into native sheets or drawers, but critical task controls must remain reachable without horizontal clipping.

Widths at or below 820px are mobile in the reference and are deliberately outside this desktop baseline. The root landing page, marketing/ad screens, mobile application, prototype data, and prototype actions are also excluded.
